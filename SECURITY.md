# Security Policy

## Supported Versions

Security updates are provided only for the latest stable release of LostNumber.

| Version | Supported          |
| ------- | ------------------ |
| 2.1.x   | :white_check_mark: |
| 2.0.x   | :x:                |
| < 2.0   | :x:                |

Users are encouraged to update to the latest available version before
reporting a security issue.

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues,
pull requests, discussions, or social media.

To report a vulnerability privately:

1. Open the **Security** tab of the LostNumber GitHub repository.
2. Select **Report a vulnerability**.
3. Provide a clear description of the issue and the steps required to
   reproduce it.

Please include, where applicable:

- the affected application version;
- the affected platform and Android version;
- detailed reproduction steps;
- the expected and actual behavior;
- screenshots, logs, or proof-of-concept code;
- the potential security impact;
- any suggested mitigation or fix.

## Response Process

After receiving a report, the project maintainer will:

- acknowledge the report within 7 days;
- review and attempt to reproduce the issue;
- provide status updates when meaningful progress is made;
- determine the severity and affected versions;
- prepare and publish a fix when the vulnerability is confirmed.

If the report is accepted, information about the vulnerability may be
published after a fix is available.

If the report is declined, the reporter will receive an explanation when
possible.

## Responsible Disclosure

Please allow reasonable time for investigation and remediation before
publishing details about a vulnerability.

Do not access, modify, delete, or disclose data belonging to other users while
testing a potential vulnerability.

Good-faith security research that follows this policy is appreciated.
